 Command-Line Options
 ----------------------

The command-line compilers BPC.EXE and TPC.EXE let you invoke all the
functions of the IDE compilers (BP.EXE, TURBO.EXE, and BPW.EXE) from the DOS
command line.

You have a choice of two command-line compilers, BPC.EXE or TPC.EXE.
  - BPC.EXE runs in DOS protected mode and targets DOS real-mode, Windows,
    and DOS protected-mode applications.
  - TPC.EXE runs only in DOS real mode and targets only DOS real-mode
    applications.

You run the command-line compilers from the DOS prompt using the following
syntax:
 BPC [options] filename [options]
 TPC [options] filename [options]

where:
  - options are zero or more optional parameters that provide additional
    information to the compiler.
  - filename is the name of the source file to compile.
     - If filename does not have an extension, the command-line compiler
       assumes .PAS.
     - If you don't want the file you're compiling to have an extension,
       append a period (.) to the end of the filename.

 TPC.EXE
For TPC.EXE, if the source text contained in filename is a program, the
compiler creates an executable file named filename.EXE, and if filename
contains a unit, the compiler creates a unit file named filename.TPU.

 BPC.EXE
For BPC.EXE, if the source text contained in filename is a program, the
compiler creates an executable file named filename.EXE, and if filename
contains a library, the compiler creates a file named filename.DLL.

If filename contains a unit, the compiler creates a unit file named
filename.TPU, filename.TPP, or filename.TPW, depending on the selected
target platform. This table illustrates the relationship between the
target platform and the unit file extension the compiler uses.

     Target platform    |  Unit file extension
     ==========================================
     DOS real mode      | .TPU
     DOS protected mode | .TPP
     Windows            | .TPW

 Options:
Here are the command-line compiler options:

 Compiler mode options:
   Option               Meaning
   /B                   Build All
   /C                   Compile
   /F                   Find Error
   /L                   Link Buffer
   /M                   Make
   /Q                   Quiet (no IDE equivalent)

 Conditional defines option:
   /D                   Conditional Defines

 Debug options:
   Option               Meaning
   /G                   .MAP File
   /V                   Debug Info in EXE Option

 Directory options:
   Option               Meaning
   /E                   EXE and TPU Directory
   /I                   Include Directories
   /O                   Object Files Directories
   /R                   Resource Directories
   /T                   Turbo Directory
   /U                   Unit Directories

Most of the command-line options have equivalent menu commands. See the
Command-Line Options and IDE Equivalents Help screen for a summary.
