 - Compile|Destination (TURBO only)
 ----------------------
With the Compile|Destination command, you
specify whether the executable code will be
stored on disk (as an .EXE file) or in memory.

Setting Destination to Disk increases the
memory available in the IDE to compile and
debug your program.

If you set Destination to Memory, the code
is lost when you exit  Turbo  Pascal.

Note: Even if Destination is set to Memory,
      any units recompiled during a Make or
      Build have their .TPU files updated on
      disk.

If you set Destination to Disk,  Turbo  Pascal
creates an .EXE file whose name is derived
from one of two names, in the following order:

 » The Primary File name
           OR
 » If no Primary File is specified, the name
   of the file in the active Edit window.

 Turbo  Pascal stores the resulting .EXE in the
same directory as the source file, or in the
Options|Directories|EXE & TPU Directory, if
one is specified.
